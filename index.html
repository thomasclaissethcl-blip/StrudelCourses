Aide audio et tempo

Si vous n’entendez rien

Commencez par la leçon « Tester le moteur audio sans sample ». Si le synthé fonctionne mais pas les batteries, attendez quelques secondes puis relancez la lecture. Les exercices à samples du module commencent par une ligne samples(...). Ne la supprimez pas. Le premier passage peut malgré tout rester silencieux quelques instants, car les fichiers audio sont chargés à la demande.

Dans le REPL intégré, il faut lancer la lecture avec le bouton Play de Strudel. Le bouton de ce module charge seulement le code dans l’interface Strudel.

Samples

Dans ce module, les samples comme bd, sd, hh et cp sont chargés explicitement depuis Dirt-Samples. La ligne à conserver est samples('https://raw.githubusercontent.com/tidalcycles/Dirt-Samples/master/strudel.json'). Elle évite de dépendre du chargement implicite de la carte de samples du REPL.

Tempo

Strudel utilise des cycles par minute. Dans ce module, quand un cycle correspond à quatre temps, un tempo de 120 BPM s’écrit setcpm(120/4). Écrire setcpm(120) avec quatre temps dans le cycle donne une sensation quatre fois trop rapide.

setcpm(120/4)
// 120 BPM perçu, 4 temps par cycle

samples('https://raw.githubusercontent.com/tidalcycles/Dirt-Samples/master/strudel.json')

setcpm(120/4)
// 120 BPM perçu, 4 temps par cycle

s("bd ~ sd ~")

Atelier libre

Collez un pattern, modifiez-le, puis chargez-le dans le REPL Strudel. L'audio s'exécute dans Strudel, l'application conserve votre code en local.
